目标不是一张静态图

这次验证的是复杂图表交互的收敛能力

行情信息x 轴是时间，y 轴是价格，hover 展示涨跌额、涨跌幅、成交量、成交额。
范围切换分时、5日、日K、周K、月K、季K、年K共用同一套窗口模型。
拖拽窗口不是平移图表，而是在固定画布里扩展或收缩 x 轴覆盖范围。

所以核心工作不是“画线”，而是把用户描述里的交互语义翻译成稳定的数据窗口。

技术选型

克制一点：先把 Demo 做成可跑、可测、可复现

React + Vite 负责页面组织，ECharts 负责图表能力，数据先用确定性模拟，避免接口、跨域和停盘影响迭代。

npm install echarts
echartsUse([

  TooltipComponent,
  CanvasRenderer

先收敛交互模型，再替换真实数据源。

拖拽不是固定窗口平移，而是在原窗口上追加数据

画布宽度不变，x 轴范围变长，视觉上被压缩
交互区域被重新定义

曲线下方扩展，曲线上方收缩

白色区：收缩窗口
填充区：扩展窗口
体验调优

边界反馈不再移动整张图，而是显示边缘阻尼

拖到头时，右侧出现灰色抛物线区域。用户知道已经到边界，但图表本体仍保持稳定。

这次分享，用一个 ECharts 行情图 Demo，复盘 vibe-coding 怎么处理复杂交互。
我们不是只做一张能看的图，而是把需求、误解、修正、测试和复现指引串起来。
起点是一张富途风格行情页：时间在 x 轴，价格在 y 轴，hover 要看到交易信息。
技术选型很克制：React 加 Vite，图表用 ECharts，数据先用确定性模拟。
第一版完成了折线、成交量、范围切换和 tooltip。但真正的难点，是拖拽窗口。
最开始我把拖拽做成固定窗口平移。用户指出，这不是要滑走终点。
正确模型是：图表宽度不变，x 轴范围变长，视觉上发生压缩。
曲线下方的面积区负责扩展窗口，曲线上方白色区域负责收回窗口。
边界反馈也调整过：整张图表不动，只在边缘显示阻尼提示。
拖拽抖动来自重新分桶和 smooth 曲线，修复方式是直接使用 raw samples。
最后整理视觉：y 轴标注、整体留白，以及红色到白色的面积渐变。
