# 视觉规范：AI Website 技术分享

## 风格定位

面向工程复盘和经验分享，整体保持清晰、克制、专业。视觉语言接近产品文档和数据看板：浅色背景、明确层级、稳定节奏，重点突出「从需求讨论到可复现 Demo」的过程感。

## 色彩

- 背景：`#fbfaf7`、`#ffffff`
- 正文：`#171717`
- 次级文字：`#667085`
- 分割线：`#e5e7eb`
- 行情上涨红：`#f0445f`
- 行情下跌绿：`#16a779`
- 工程蓝灰：`#475467`
- 柔和强调底色：`#fff1f3`、`#edf7f3`、`#eef4ff`

## 字体

中文优先使用系统无衬线字体：

```css
font-family: "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
```

代码、提交号、时间戳使用等宽字体：

```css
font-family: "SFMono-Regular", ui-monospace, Menlo, Consolas, monospace;
```

## 构图

- 视频默认使用 `1920 x 1080` 横屏画布。
- 每屏只讲一个核心判断，不堆叠过多卡片。
- 图表截图和交互示意优先占据主视觉区域。
- 标题使用大字号但不夸张，正文保持可扫读。
- 字幕位于底部安全区，不遮挡截图主体。

## 动效

- 以轻量 push slide 和 blur crossfade 为主。
- 每个场景有入场动画，避免静态跳切。
- 不使用炫技型 glitch、霓虹、强透视或高频闪烁。
- 关键概念用线条、标注、局部放大和流程箭头表达。

## 不做什么

- 不使用深色赛博、紫蓝渐变或纯营销落地页风格。
- 不把内容塞进多层卡片。
- 不用过小文字解释复杂代码。
- 不用没有上下文的装饰图形替代真实 demo 截图。
